Java 中常见的命名约束

本文指出的是 Java 常见命名中比较统一的约定俗成的词缀

词缀 含义 来源
Abstracr / Base 抽象 JDK
Enum /Type 枚举 JDK
Utils 工具 JDK
Exception 异常 JDK
Impl 接口实现 JDK
DO（ Data Object） 数据对象，与数据库表结构一一对应，通过DAO层向上传输数据源对象 领域模型相
DTO（ Data Transfer Object） 数据传输对象，Service或Manager向外传输的对象 领域模型相
AO（ Application Object） 应用对象，在Web层与Service层之间抽象的复用对象模型 领域模型相
BO（ Business Object） 业务对象，由Service层输出的封装业务逻辑的对象 领域模型相
VO（ View Object） 显示层对象，通常是Web向模板渲染引擎层传输的对象 领域模型相
POJO（ Plain Ordinary Java Object） 只有setter/getter/toString的简单对象，通常可以包括以上几种 领域模型相
Query 数据查询对象 领域模型相
Handler 异步的回调函数（子程序）
Predicate 断言
Validator 校验器
Test 测试
Controller 控制层 MVC 分层
Service 服务层 MVC 分层
ServiceImpl 服务实现层 MVC 分层
DAO 数据操作层 MVC 分层
is 对象是否符合期待的状态 返回真伪值
can 对象能否执行所期待的动作 返回真伪值
should 推荐还是不推荐 返回真伪值
has 对象是否持有 返回真伪值
needs 是否需要 返回真伪值
ensure 检查是否为期待的状态 用来检查的方法
validate 检查是否为正确的状态 用来检查的方法
IfNeeded 需要的时候执行，不需要则什么都不做 按需求执行的方法
might 需要的时候执行，不需要则什么都不做 按需求执行的方法
try 尝试执行，失败时抛出异常或返回errorcode 按需求执行的方法
OrDefault 尝试执行，失败时返回默认值 按需求执行的方法
OrElse 尝试执行，失败时返回实际参数中指定的值 按需求执行的方法
force 强制尝试执行，error抛出异常或是返回值 按需求执行的方法
blocking 线程阻塞方法 异步相关方法
InBackground 在后台执行线程 异步相关方法
Async 异步方法 异步相关方法
Sync 同步方法 异步相关方法
schedule Job和Task放入队列 异步相关方法
post Job和Task放入队列 异步相关方法
on 事件发生时执行 回调方法
before 事件发生前执行 回调方法
pre 事件发生前执行 回调方法
will 事件发生前执行 回调方法
after 事件发生后执行 回调方法
post 事件发生后执行 回调方法
did 事件发生后执行 回调方法
should 确认事件是否可以执行 回调方法
initialize 初始化或延迟初始化 操作对象声明周期
pause 暂停 操作对象声明周期
stop 停止 操作对象声明周期
abandon 销毁 操作对象声明周期
destory 销毁 操作对象声明周期
dispose 销毁 操作对象声明周期
contains 包含指定对象 集合操作
add 添加 集合操作
append 添加 集合操作
insert 插入到指定index位置 集合操作
put 添加到指定key位置 集合操作
remove 移除元素 集合操作
enqueue 添加到队列尾 集合操作
dequeue 从队列头部取出并移除 集合操作
push 添加到栈头 集合操作
pop 从栈头取出并移除 集合操作
peek 从栈头取出不移除 集合操作
find 找到符合项 集合操作
create 创建 数据相关方法
new 创建 数据相关方法
from 从已有的某物创建 数据相关方法
to 转换 数据相关方法
update 更新 数据相关方法
load 读取 数据相关方法
fetch 远程读取 数据相关方法
delete 删除 数据相关方法
remove （通常是单对象）删除 数据相关方法
save 保存 数据相关方法
store 保存 数据相关方法
commit 提交 数据相关方法
apply 应用，使生效 数据相关方法
clear 清除 数据相关方法
reset 重置或恢复到初始状态 数据相关方法
